# Claude Design ワークフロー — Anthropicインサイダーの7ステップ

## Claude Designとは何か

Claude Designは「AIにデザインを頼むキャンバスツール」ではなく、**Claude Codeと同じ"生き物"として扱うべきツール**。

命令して待つのではなく、**対話しながら育てる**感覚が近い。このスキルでは、Anthropicの内部デザイナーが実践する7ステップワークフローをガイドする。

## 7ステップワークフロー

### Step 1: デザインシステム + コア画面を先に整える

最初にアクションを起こす前に、まず「基盤」を作る。

```
1. カラーパレット（プライマリ/セカンダリ/ニュートラル）を決める
2. タイポグラフィスケール（見出し/本文/ラベル）を定義する
3. 最も重要な1〜2画面のコア構造を作る
```

**やりがちな失敗**: ページを一気に全部作ろうとする。基盤なしで作ると後からの一貫性確保が大変になる。

---

### Step 2: エンジニアとライブイテレーション

Claude Designはエンジニアと「同時に」作業するためのツール。

- エンジニアがコードを見ながら、デザイナーがClaude Design上でリアルタイムに変更
- 「このコンポーネントのここを変えて」→ 即反映 → コードに取り込む
- フィードバックループを短くする

---

### Step 3: コメントツールで精密編集

Claude Designの「コメント機能」は単なるメモではない。**精密な編集指示ツール**として使う。

```
❌ 「全体的にもっとシンプルにして」
✅ 「[コメント]このボタンの角丸を4pxから2pxに。テキストを12pxに」
```

具体的な数値や場所を指定することで、Claudeが迷わず実行できる。

---

### Step 4: ビデオデモを生成する

静的なスクリーンショットではなく、**動くデモ**を作る。

Claude Designでプロトタイプが完成したら：
1. インタラクションを付ける（ホバー/クリック/トランジション）
2. 画面収録またはGIF化
3. ピッチ・提案書・SNSに活用

---

### Step 5: コネクターを活用する（Docs/Slack）

Claude DesignをDocsやSlackと接続することで、デザインに「文脈」を持たせる。

- **Docs連携**: PRD・仕様書・ブランドガイドラインを参照させる
- **Slack連携**: チームのフィードバックをリアルタイムに反映

ドキュメントなしで作ると「なんとなくそれっぽいもの」になる。文脈を持たせることで「目的を持ったデザイン」になる。

---

### Step 6: カスタムツールを作成する

繰り返す作業はカスタムツール化する。

```
例：
- 「ヘッダーコンポーネントを生成するツール」
- 「このブランドカラーでカードUIを作るツール」
- 「モバイル版に変換するツール」
```

3回同じことを頼んでいたら、ツール化のサイン。

---

### Step 7: スローダウンすべきタイミングを知る

Claude Designが苦手なことを知っておくことが、最終的な完成度を上げる鍵。

**AIに任せずに手作業すべきもの**:
- アイコンの選定・調整（Lucide/Heroiconsから手動選択）
- ブランドの固有名詞・製品名の命名
- 極めて細かいピクセル調整（1〜2pxのズレ）
- 最終的なコピーライティング

「AIに頼みすぎない判断力」がClaude Design達人の条件。

---

## 実践ガイド：スタートアップのピッチ資料UI

### 典型的なワークフロー例

```
1. [Step 1] 「カラーはネイビー×ホワイト、フォントはInter、メイン画面はダッシュボード」と伝える
2. [Step 3] コメントツールで「ここのKPIカードを3列に、数字を大きく」と精密指示
3. [Step 2] エンジニアと同期しながらコンポーネントを調整
4. [Step 4] 画面遷移デモを録画してSNSに投稿
5. [Step 7] ロゴとコピーは自分で最終調整
```

---

## よくある失敗パターン

| 失敗 | 原因 | 解決策 |
|------|------|--------|
| 毎回違うデザインになる | Step 1をスキップ | デザインシステムを先に定義する |
| Claudeが意図を理解しない | 抽象的な指示 | Step 3のコメントツールで数値指定 |
| 「AIっぽい」見た目になる | ありきたりなプロンプト | 参考サイトのURLや実例画像を渡す |
| 細部が雑になる | 全部AIに任せる | Step 7のスローダウン判断 |
