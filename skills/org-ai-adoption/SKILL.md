# Org AI Adoption

組織全体へのAI導入プロセスを体系化し、実行可能な推進計画をMarkdownで出力するスキルです。

## このスキルが解決すること

「会社でAIを使いたいけど、何から始めればいいかわからない」「導入はしたが定着しない」「推進担当者になったが計画の立て方がわからない」——そんな組織レベルの悩みに応えます。

個人のAI活用スキルアップではなく、**組織として**AIを導入・定着・拡大させるためのフレームワークと実行計画を生成します。

## ワークフロー

### Step 1: 組織コンテキストの収集

以下の情報をユーザーに確認する（わかる範囲でOK）:

1. **会社規模・業種**: 何人くらいの組織？どんなビジネスか？
2. **現状のAI活用レベル**: まったく使っていない / 一部の人が個人利用 / 組織的に取り組み始めている
3. **主な課題・ペインポイント**: 何を解決したいのか？（業務効率化・コスト削減・新規事業など）
4. **推進体制**: AI推進担当者はいるか？経営のコミットメントはあるか？
5. **制約・懸念**: セキュリティ・コンプライアンス要件、予算感、反対意見など
6. **タイムライン**: いつまでに何を実現したいか？

情報が不足している場合は合理的に補完して進める。

### Step 2: 現状アセスメント

収集した情報をもとに、組織のAI成熟度を以下の4段階で評価する:

| レベル | 状態 | 特徴 |
|--------|------|------|
| L1 | 認知段階 | AIツールの存在は知っているが、業務利用ゼロ |
| L2 | 実験段階 | 一部の先進的な社員が個人的に試している |
| L3 | 組織化段階 | チーム・部門単位での利用が始まっている |
| L4 | 定着段階 | 全社的な標準プロセスとしてAIが組み込まれている |

現状レベルを明示し、次のレベルへの移行計画を中心に設計する。

### Step 3: 推進計画の生成

以下の4つのコンポーネントをMarkdownで出力する:

---

#### A. キックオフワークショップ設計

```markdown
## ワークショップ設計

**目的:** [組織の現状に合わせた目的]
**対象:** [参加推奨メンバー]
**所要時間:** [推奨時間]

### アジェンダ
1. [セクション名] — [時間] — [内容・狙い]
2. ...

### 事前準備
- [参加者への事前課題]
- [用意するもの]

### 期待されるアウトプット
- [ワークショップ終了時に決まっていること]
```

---

#### B. 評価基準（KPI設計）

```markdown
## AI導入の評価基準

### 短期KPI（3ヶ月）
- [測定可能な指標]: [目標値]

### 中期KPI（6ヶ月）
- [測定可能な指標]: [目標値]

### 長期KPI（1年）
- [測定可能な指標]: [目標値]

### 定性的な評価項目
- [社員の意識・スキル変化]
- [文化的変化の兆候]
```

---

#### C. 全社展開ロードマップ

```markdown
## 展開ロードマップ

### Phase 1: パイロット（〜Nヶ月）
**ゴール:** [フェーズゴール]
**対象部門/チーム:** [先行部門の選定基準]
**主なアクション:**
- [ ] [アクション]
- [ ] [アクション]
**成功条件:** [次フェーズに進む判断基準]

### Phase 2: 横展開（〜Nヶ月）
...

### Phase 3: 定着・高度化（〜Nヶ月）
...
```

---

#### D. 推進体制・役割定義

```markdown
## 推進体制

### コアチーム
| 役割 | 責任 | 推奨スキル |
|------|------|-----------|
| AI推進リード | ... | ... |
| 部門チャンピオン | ... | ... |
| IT/セキュリティ担当 | ... | ... |

### エスカレーションフロー
[課題発生時の判断・承認フロー]
```

---

### Step 4: よくある失敗パターンと対策

組織規模・業種・現状に応じた「このパターンで詰まりやすい」という注意点を3〜5個提示する:

例:
- **「推進者だけ熱心で現場が動かない」パターン** → 現場の小さな成功事例を早期に可視化する
- **「とりあえず研修したけど定着しない」パターン** → 業務フローへの組み込みが必要
- **「セキュリティが怖くて進まない」パターン** → 情報分類ルールを先に整備する

### Step 5: ネクストアクション（Today/This Week）

抽象的なロードマップだけでなく、**明日から動ける具体的なアクション**を3点に絞って提示する:

```markdown
## 今すぐ始めるアクション

1. **Today:** [今日やること — 30分以内で完了できるもの]
2. **This Week:** [今週中にやること — 小さく確実に]
3. **This Month:** [今月中に達成したいマイルストーン]
```

## 出力形式

最終成果物として以下を出力する:

1. **`ai-adoption-plan.md`** — 推進計画フルドキュメント（保存推奨）
2. **エグゼクティブサマリー** — 経営層への説明用に1ページ分の要約
3. **ネクストアクション** — 明日から動ける3ステップ

## 補足・カスタマイズ

- **業種別の考慮点**: 金融・医療はコンプライアンス重視、製造業はオペレーション改善から入るのが効果的など
- **規模別のアプローチ**: 10人以下のスタートアップと1000人超の大企業では推進戦略が異なる
- **ツール選定支援**: Claude/ChatGPT/Copilotなどのツール比較が必要な場合は追加で対応可能
- **社内説明資料の作成**: 経営層向け・現場向けの説明文や資料の骨子も生成できる
