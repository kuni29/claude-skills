# 見積書作成スキル

Algomatic社フォーマットの見積書PDFを生成する。
金額計算・備考フォント調整はスクリプトが自動処理するため、Claude はパラメータ収集に集中する。

## 起動時ガイダンス

スキルがトリガーされたら、作業を始める前に以下のガイダンスをユーザーに表示する:

---

**見積書を作成します。**

まず、どの方法で始めますか？

- **画像を貼る** — 既存の見積書やスライドの画像から再現します
- **対話で入力** — 質問に答える形で順番に情報を入力します
- **YAMLファイル指定** — 既存のパラメータファイルがあればそれを使います
- **既存見積の修正** — 以前作った見積書の日付や項目を変更します

この後の流れ: パラメータ収集 → YAML生成 → PDF自動生成 → 確認・修正
（金額計算・レイアウト調整は全て自動です）

---

ガイダンスを表示した後、ユーザーの選択に応じて Step 1 の該当モードに進む。
ユーザーが既に画像やYAMLを添付している場合、または具体的な指示がある場合はガイダンスを省略して直接該当モードに入る。

## ワークフロー

### Step 1: パラメータ収集

入力モードに応じてパラメータを収集する。

#### モードA: 画像入力
見積書・提案スライド等の画像を Read ツールで読み取り、金額・項目・日付・備考を抽出する。
抽出したパラメータをユーザーに一覧表示し、間違いや不足がないか確認してから次に進む。

#### モードB: 対話入力（AskUserQuestion）
以下の順序で AskUserQuestion を使い、1問ずつ聞いていく。
前の回答から推測できる項目はデフォルト値を提示し、ユーザーが Enter だけで進めるようにする。

**基本情報:**
1. 宛先企業名（例: 株式会社サンプル商事）
2. 案件名
3. 見積日 / 見積No.
4. 契約期間（開始日〜終了日）
5. 検収日 or 納品日
6. 支払条件（デフォルト: 当月末締翌月末払い）
7. 有効期限（デフォルト: 見積日から1ヶ月）

**明細行:**（繰り返し）
8. 項目名 / 数量 / 単位 / 単価（「完了」と入力されるまで繰り返す）

**オプション:**
9. 出精値引の有無と金額・理由
10. 備考（改行区切りで入力、または「デフォルト」で標準備考を使用）

最後に入力内容の一覧を表示し、確認を取る。

**標準備考**（「デフォルト」選択時）: `references/sample_params.yaml` の notes をそのまま使用する。

#### モードC: YAML直接指定
既存の YAML ファイルを読み込む。`references/params_schema.md` に従っているか検証し、不足フィールドがあれば AskUserQuestion で補完する。

#### モードD: 既存見積の変更
既存 YAML をコピーし、変更したいフィールドを AskUserQuestion で確認する。変更点のみ聞く。

### Step 2: YAMLパラメータファイル作成

`references/params_schema.md` のスキーマに従い YAML ファイルを作成する。
サンプルは `references/sample_params.yaml` を参照。

出力先は `output/estimates/` ディレクトリに保存する。

会社情報（Algomatic）はデフォルト値があるためYAMLでは省略可能。
`amount` フィールドも省略可能（`quantity × unit_price` で自動計算）。

YAMLを書き出した後、ユーザーに内容を表示して最終確認を取る。

### Step 3: PDF生成

以下のコマンドを実行:

```bash
mkdir -p output/estimates
uv run <this-skill-path>/scripts/generate_estimate.py <params.yaml> output/estimates/<output.pdf>
```

HTML ファイルも同名で自動保存される（デバッグ・微調整用）。

### Step 4: 確認

生成された PDF を Read ツールで表示してユーザーに確認する。

**チェックポイント:**
- 1ページに収まっているか
- 金額計算が正しいか
- 宛先・日付に誤りがないか

修正が必要な場合は YAML を更新して Step 3 を再実行する。

### Step 5: ファイル名整形

最終版のファイル名を先方向けに整形する。
命名規則: `【Algomatic】お見積書_<案件名>.pdf`

YAML の `output.filename` に指定があればそれを使用する。

## 日付変更版の作成

請求書同日付版など、日付だけ変えたバリエーションが必要な場合:

1. 既存 YAML をコピー
2. `estimate.date` と `terms.valid_until` を変更
3. 備考に経緯の注記を追加（例: 「本見積書は YYYY年MM月DD日にご提示した提案内容に基づき、正式書面として発行するものです。」）
4. ファイル名にサフィックスを追加（例: `_請求書同日付版`）

## 注意事項

- 金額は全てスクリプトが自動計算する。YAML の `amount` を手動設定するのは端数調整が必要なケースのみ
- 備考の行数に応じてフォントサイズが 8.5pt〜7pt で自動調整される
- エンタープライズ向け文書のため、丁寧語・正式表記を使う
