# Claude Code コンテキストマネージャー

## なぜコンテキスト管理が重要か

Claude Codeは100万トークンのコンテキストウィンドウを持っているが、**30〜40万トークン付近からContext Rot（コンテキスト腐敗）が発生**し始める。症状は：

- 同じ間違いを繰り返す
- 以前確認したことを「忘れた」かのように振る舞う
- 回答の質が下がる・一貫性がなくなる
- 指示を部分的にしか守らなくなる

これはモデルの能力の問題ではなく、コンテキストの使い方の問題。適切なタイミングで適切なアクションを取ることで防げる。

## 診断フロー

ユーザーにコンテキスト管理の相談をされたら、以下の質問を心の中でチェックし、状況を診断して推奨アクションを提示する：

1. **直前のターンで失敗・混乱があったか？** → `rewind`
2. **作業は続けたいがコンテキストが膨らんでいるか？** → `compact`
3. **今のタスクは完全に終わり、新しいことを始めるか？** → `clear`
4. **独立して並行処理できるサブタスクがあるか？** → `subagents`
5. **特に問題なし？** → `continue`

## 5つのアクション詳細

### 1. continue（継続）
**使う場面**: 品質が維持されており、まだコンテキストに余裕がある

特別な操作不要。ただしこまめに状態を確認することを推奨。

---

### 2. rewind（巻き戻し）★最も過小評価されている機能
**使う場面**: 直前のやりとりが失敗した・混乱した・無駄なトークンを消費してしまった

**操作方法**: `Esc` を2回押す（`Esc×2`）

**効果**: 失敗した試行のトークンをコンテキストから切り捨てて、なかったことにして再挑戦できる。`/clear` と違い、それ以前の有効な文脈は保持される。

**こんなときに使う**:
- 「あ、違う方向に行ってしまった」と気づいたとき
- 長い失敗回答が生成された直後
- ループに入ってしまったとき

---

### 3. compact（要約圧縮）
**使う場面**: 作業は続けたいが、コンテキストが膨らんできた

**操作方法**: `/compact` コマンドを実行

**重要**: `compact` は方向性の指示を添えると効果が高い。方向性なしで実行すると汎用的な要約になり、次のタスクに必要な情報が失われることがある。

**方向性指示テンプレート**:
```
/compact 以下の情報を優先して保持してください：
- [今後のタスクで必要な具体的情報]
- [重要な決定事項・制約]
- [次に取り組む作業の文脈]
```

**例**:
```
/compact 以下を優先保持：
- 作成中のスキルのSKILL.mdドラフト
- ユーザーからのフィードバック（簡潔さ重視・コメント不要）
- 次のステップ：テストケース3本を作成してeval実行
```

---

### 4. clear（全リセット）
**使う場面**: タスクが完全に終わり、全く新しいことを始めるとき

**操作方法**: `/clear` コマンドを実行

**効果**: コンテキストを完全にクリア。新鮮な状態でスタートできる。

**注意**: 継続中の作業が失われるため、必要な情報は事前にファイルやメモに保存してから実行する。

---

### 5. subagents（並行エージェント）
**使う場面**: 独立した複数のサブタスクを並行処理したいとき

**効果**: 各サブエージェントが独自のコンテキストウィンドウを持つため、メインの会話が重くならない。

**向いているケース**:
- 複数ファイルの独立した調査・作成
- A/Bテスト的な比較作業
- 大規模なコードベースの並列分析

---

## Context Rot の早期サイン

以下が見られたら、コンテキスト管理を見直すタイミング：

| サイン | 推奨アクション |
|--------|---------------|
| 同じ間違いを2回以上繰り返している | `rewind` または `compact` |
| 「さっき言ったこと」を忘れているような回答 | `compact`（方向性指示付き） |
| 回答が長くなりすぎている・冗長 | `compact` |
| 全く関係ない内容が混入してきた | `clear` |
| 複数のタスクが混在して混乱している | `clear` + 個別に再開 |
| タスクが完了した | `clear` |

## 推奨：定期チェックのタイミング

- **10ターンごと**: 品質チェック
- **20ターン**: compact を検討
- **30ターン以上**: 必要なら clear を強く推奨
- **タスク完了時**: 必ず clear
