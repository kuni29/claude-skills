テンプレートスライド(template/template.pptx)は全11枚で構成されている。
これらは「そのままコピーしてテキストを差し替える」ためのものではなく、企業のスライドフォーマット（配色・フォント・余白・装飾スタイル）を引き継ぎつつ、内容に応じてレイアウトを新規構築するための参考例である。

## スライド1: 表紙（カバースライド）

**レイアウト概要:** 背景画像の上に、宛先（サブタイトル）・プロジェクト名と資料種別（メインタイトル）・会社ロゴを左寄せで配置。

**構成要素:**

- サブタイトル: 宛先（「〇〇株式会社 御中」）
- メインタイトル: プロジェクト名 + 資料種別（改行区切り）
- ロゴ: 左下に ALGOMATIC ロゴ

**記載内容の例:**

- 「株式会社ABC 御中」
- 「ECサイトリニューアルプロジェクト」+「キックオフ資料」
- 「AI導入PoC」+「第3回 進捗報告」

**調整の例:**

- 資料種別が長い場合、メインタイトルを2行＋資料種別を3行目にして、フォントサイズを縮小（36pt → 28pt）
- 日付や版番号を追加したい場合、ロゴの上にテキストボックスを追加（例: 「2026年2月20日 v1.0」）
- 社外向け/社内向けで宛先の有無を切り替え（社内向けなら宛先を削除してタイトルを上に移動）

## スライド2: 振り返りと目的（2カラム比較）

**レイアウト概要:** タイトル＋メインメッセージの下に、左右2カラムで「これまで」と「本日の目的」を対比する。各カラムはダークネイビーの見出しバー＋ライトブルーの箇条書きエリア。

**構成要素:**

- タイトル: スライドの主題
- サブタイトル: 1行で要約するメインメッセージ
- 左カラム見出し（dk2背景・白文字）+ 左カラム本文（accent3背景・箇条書き）
- 右カラム見出し（dk2背景・白文字）+ 右カラム本文（accent3背景・箇条書き）

**記載内容の例:**

- タイトル: 「これまでの振り返りと本日の目的」
- メインメッセージ: 「要件定義を完了し、本日は設計方針の合意を目指します」
- 左カラム「これまで」: 「● 業務フロー分析完了」「● 要件一覧をレビュー済み」「● 技術選定の候補を3案に絞込」
- 右カラム「本日の目的」: 「● システム構成案の合意」「● 開発スケジュールの確認」「● 次回までのタスク割り振り」

**調整の例:**

- 項目数が片方だけ多い場合、多い方のカラム幅を広く取る（例: 左60%、右40%）
- 3カラム化して「これまで / 本日 / 次回」の時系列比較にする場合は、各カラム幅を均等3分割に変更
- 箇条書きの代わりにアイコン＋テキスト行にする場合、箇条書きの「●」を削除し、行頭にアイコン図形を配置

## スライド3: アジェンダ

**レイアウト概要:** 左1/3にダークネイビー背景＋白文字で「Agenda」、右2/3に番号付きのアジェンダ項目を縦に並べる。

**構成要素:**

- 左パネル: 「Agenda」ラベル（大きな白文字、ダークネイビー背景）
- 右パネル: 番号＋アジェンダ項目のリスト（「01 〇〇」「02 〇〇」形式）

**記載内容の例:**

- 「01 プロジェクト概要」
- 「02 現状分析と課題」
- 「03 提案するソリューション」
- 「04 スケジュールとコスト」
- 「05 次のステップ」

**調整の例:**

- アジェンダが3項目なら余白を大きく取って読みやすくする（行間を広げる）
- アジェンダが6項目以上ある場合はフォントサイズを20pt → 16ptに縮小し、行間を詰める
- 各項目にサブテキスト（補足説明）を付ける場合、番号+タイトル行の下に12ptのグレー文字で1行追加

## スライド4: セクション区切り

**レイアウト概要:** スライド3と同様の左右分割。左パネルに大きなセクション番号（「01」）、右パネルにセクション名。

**構成要素:**

- 左パネル: セクション番号（超大文字、白、ダークネイビー背景）
- 右パネル: セクション名（大きなダーク文字）

**記載内容の例:**

- 左: 「01」 → 右: 「プロジェクト概要」
- 左: 「02」 → 右: 「現状の課題分析」

**調整の例:**

- セクション名が長い場合（例:「AI活用による業務効率化の提案」）、右パネルのフォントサイズを下げる（44pt → 32pt）か、2行に折り返す
- セクション番号の下にサブテキスト（英語表記等）を追加する場合、番号の下に16ptで配置（例: 「01」の下に「Overview」）
- セクション内容を示唆するキーワードを右パネルに小さく追記（例: セクション名の下に「〜背景・目的・スコープ〜」）

## スライド5: 課題提起（3カラム＋強調フッター）

**レイアウト概要:** タイトル＋メインメッセージの下に、等幅3カラムのプレースホルダー（画像/図/テキスト用）を横並び。スライド最下部にフルワイド（スライド幅いっぱい）のダークネイビーバーで強調メッセージを配置。

**構成要素:**

- タイトル + サブタイトル（メインメッセージ）
- 3つの等幅コンテンツボックス（白背景・ダークネイビー枠線）
- フルワイドの強調メッセージバー（dk2背景・白文字）

**記載内容の例:**

- タイトル: 「現状の課題」
- メインメッセージ: 「3つの領域で業務効率に重大なボトルネックが発生しています」
- ボックス1: 受注処理のフロー図 + 「手作業が全体の60%を占める」
- ボックス2: エラー率の推移グラフ + 「月間エラー件数が前年比1.5倍」
- ボックス3: コスト構造の円グラフ + 「人件費が予算の80%に到達」
- 強調メッセージ: 「このまま放置すると、年間約3,000万円の機会損失が見込まれます」

**調整の例:**

- 課題が2つの場合は2カラムに変更し、各ボックスの幅を広げる（2787000 → 4210200 EMU）
- 各ボックス内に画像ではなくテキストを入れる場合、ボックスをaccent3背景に変更して文字の読みやすさを確保
- 強調メッセージが不要な場合はフッターバーを削除し、コンテンツボックスの高さを拡張

**3つのコンテンツボックスの扱い:**

各ボックスは本来「図・グラフ・画像」を配置するためのプレースホルダーであり、長文テキストの流し込みには向かない。内容に応じて図形グリッドや画像に差し替えるか、テキスト主体であればaccent3背景に変更した上で構造化した箇条書きを配置すること。

## スライド6: 1トピック深掘り（メイン図解＋ABC解説）

**レイアウト概要:** タイトル＋メインメッセージの下を左右に分割。左側（約60%幅）にインフォグラフィックス見出しバー＋大きな図解エリア。右側（約40%幅）に解説見出しバー＋A/B/Cの3段解説ボックス（各ボックスの左に丸バッジ付き）。

**構成要素:**

- タイトル + サブタイトル
- 左: ダークネイビー見出しバー（図解のタイトル）+ 大きな白背景コンテンツエリア
- 右: ダークネイビー見出しバー（「解説 or 主張」）+ A/B/C の3段ボックス（各行にellipseバッジ + accent3背景テキスト）

**記載内容の例:**

- タイトル: 「提案するシステム構成」
- メインメッセージ: 「クラウドネイティブな3層アーキテクチャで拡張性と保守性を両立」
- 左見出し: 「システム構成図」→ 左エリア: アーキテクチャ図（画像として挿入）
- 右見出し: 「構成のポイント」
  - A: 「フロントエンドはNext.jsでSSR対応、CDN配信で高速化」
  - B: 「APIレイヤーはコンテナ化し、オートスケーリングに対応」
  - C: 「データ層はマネージドDBで運用負荷を最小化」

**調整の例:**

- 解説ポイントが2つの場合、A/Bの2段に減らしてボックスの高さを拡張（851700 → 1300000 EMU程度）
- 解説ポイントが4つ以上ある場合、各ボックスの高さを縮小するか、左右の比率を50:50に変更
- 図解エリアにテキストで説明を書く場合、白背景をaccent3背景に変更して視覚的に統一

**左側コンテンツエリア（図解エリア）の扱い:**

左側の大きなコンテンツエリアは本来「図解・インフォグラフィックス」を配置する場所であり、プレーンテキストをそのまま流し込むべきではない。内容に応じて以下の方針で対応すること。

- **構造化データ（内訳・比較表など）**: コンテンツエリアを削除し、矩形シェイプのグリッド配置（スライド9の表パターンを参考）に差し替える
- **図解・フロー図・チャート**: 画像として挿入するのが最適。画像がない場合はユーザーに提案する
- **どうしてもテキストのみ**: コンテンツエリアの背景をaccent3に変更し、箇条書きを構造化して配置するか、このレイアウト自体の採用を見直す

## スライド7: 2トピック比較

**レイアウト概要:** タイトル＋メインメッセージの下に左右2カラム。各カラムはダークネイビー見出しバー＋白背景コンテンツエリア（図/画像用）＋accent3背景の解説ボックスの3段構成。

**構成要素:**

- タイトル + サブタイトル
- 各カラム: 見出しバー（dk2背景）→ コンテンツエリア（白背景・dk2枠線）→ 解説ボックス（accent3背景）

**記載内容の例:**

- タイトル: 「現行システムと提案システムの比較」
- メインメッセージ: 「クラウド移行により運用コスト40%削減を見込みます」
- 左カラム「現行システム（オンプレミス）」: サーバー構成図 + 「月額運用費: 500万円、障害対応: 平均4時間」
- 右カラム「提案システム（クラウド）」: クラウド構成図 + 「月額運用費: 300万円、障害対応: 平均30分」

**調整の例:**

- 図解が不要で文章比較のみの場合、コンテンツエリアを削除し解説ボックスを高さいっぱいに拡張
- Before/After比較で右カラムを強調したい場合、右カラムの枠線を太く（12700 → 25400 EMU）するか、背景色を変えて目立たせる
- 各カラムの解説ボックス内に箇条書きを入れる場合、テキストの左インデントを設定しbulletを追加

**コンテンツエリア（白背景・枠線付き）の扱い:**

各カラムの白背景エリアは図解・画像用のプレースホルダー。テキストのみで比較する場合はコンテンツエリアを削除して解説ボックスを拡張し、構造化データがある場合は図形グリッドに差し替えること。

## スライド8: 3トピック並列

**レイアウト概要:** スライド7の3カラム版。各カラムの構成は同じ（見出しバー＋コンテンツエリア＋解説ボックス）。

**構成要素:**

- タイトル + サブタイトル
- 3カラム（各2787000 EMU幅）: 見出しバー → コンテンツエリア → 解説ボックス

**記載内容の例:**

- タイトル: 「3つのソリューション案」
- メインメッセージ: 「コスト・導入速度・拡張性の観点から3案を比較します」
- カラム1「案A: フルスクラッチ」: 構成イメージ図 + 「自由度は高いが開発期間12ヶ月」
- カラム2「案B: SaaS活用」: 構成イメージ図 + 「即導入可能、カスタマイズに制約あり」
- カラム3「案C: ハイブリッド」: 構成イメージ図 + 「コアはSaaS＋独自UIで6ヶ月導入」

**調整の例:**

- 推奨案を強調したい場合、該当カラムの見出しバー色を変える（例: accent1に変更）か、枠線を太くする
- 図解なしでテキスト比較のみの場合、コンテンツエリアと解説ボックスを統合して1つの大きなテキストエリアに
- 4案以上を提示する場合は、このレイアウトは不向き。表パターン（スライド9）の利用を検討

**コンテンツエリア（白背景・枠線付き）の扱い:**

スライド7と同様。各カラムの白背景エリアは図解・画像用。テキストのみの場合はコンテンツエリアと解説ボックスを統合し、構造化データは図形グリッドに差し替えること。

## スライド9: 表パターン（マトリクス比較）

**レイアウト概要:** タイトル＋メインメッセージの下に、矩形シェイプで構成した3行×3列のマトリクス。行ヘッダ（左端、dk2背景）＋列ヘッダ（上端、dk2背景）＋データセル（accent3背景）。ネイティブテーブル要素ではなく、個別の矩形シェイプをグリッド配置で実現。

**構成要素:**

- タイトル + サブタイトル
- 列ヘッダ × 3（dk2背景・白文字）
- 行ヘッダ × 3（dk2背景・白文字）
- データセル × 9（accent3背景）

**記載内容の例:**

- タイトル: 「ソリューション比較表」
- メインメッセージ: 「3案を5つの評価軸で比較した結果、案Cを推奨します」
- 列ヘッダ: 「導入コスト」「運用コスト」「拡張性」
- 行ヘッダ: 「案A: フルスクラッチ」「案B: SaaS」「案C: ハイブリッド」
- データセル例: 「3,000万円」「月額50万円」「◎ 自由に拡張可」

**調整の例:**

- 列を4つに増やす場合、各セルの幅を縮小（2082900 → 1550000 EMU程度）し、4列分のヘッダとセルを追加
- 行を増やす場合、各セルの高さを縮小（829200 → 600000 EMU程度）して行を追加
- 特定セルを強調したい場合、該当セルの背景色を変更（例: accent3 → 薄い緑で「◎」を目立たせる）
- セル内に長いテキストが入る場合、フォントサイズを14pt → 11ptに縮小して収める

## スライド10: スケジュール（ガントチャート風）

**レイアウト概要:** タイトルの下に、月別の列ヘッダ＋タスクカテゴリの行ヘッダ＋矢印（homePlateジオメトリ）によるタスクバーで構成されたガントチャート風レイアウト。タスクバーの幅と開始位置を調整することで期間を表現。

**構成要素:**

- タイトル（サブタイトルなし）
- 列ヘッダ × 3（dk2背景・白文字、月名）
- 行ヘッダ × 3（dk2背景・白文字、タスクカテゴリ名）
- タスクバー × 複数（homePlate形状、accent3背景、タスク名テキスト入り）

**タスクバーの期間表現:**

- 1列分（1ヶ月）: cx=2082900
- 2列分（2ヶ月）: cx=4256700（2列幅 + 列間ギャップ）
- 3列分（3ヶ月）: cx=6430200（3列幅 + 列間ギャップ×2）
- 開始位置を列ヘッダのx座標に合わせることで開始月を制御

**記載内容の例:**

- タイトル: 「開発スケジュール」
- 列ヘッダ: 「4月」「5月」「6月」
- 行1「要件定義・設計」: タスクバー「要件定義」（4月〜5月にまたがる）
- 行2「開発・テスト」: タスクバー「フロントエンド開発」（4月〜5月）＋「結合テスト」（6月）
- 行3「移行・リリース」: タスクバー「データ移行準備」（4月）＋「移行リハーサル」（5月）＋「本番リリース」（6月）

**調整の例:**

- 月数を4〜6ヶ月に増やす場合、列幅を縮小して列数を追加（各列幅を1500000 EMU程度に）
- 週単位のスケジュールにする場合、列ヘッダを「W1」「W2」...に変更し、バー幅を細かく調整
- マイルストーンを追加する場合、特定位置にダイヤモンド形状（diamond prst）を追加配置
- タスクカテゴリが4行以上必要な場合、行ヘッダとタスクバーの高さを縮小（829200 → 600000 EMU）
