# Claude Cowork Slides

Claude Coworkを使って「AIらしさを排除した、再現性の高いスライド」を量産するワークフロー。

## このスキルが解決する問題

- 毎回レイアウトが変わってしまう
- 文字が多くなりすぎる
- グラデーションや丸角だらけの「AI感」が出てしまう
- 1枚ずつ指示するのが面倒

---

## ワークフロー概要

```
音声メモ / 短いテキスト
        ↓
[Step 1] コンテンツ構造化
        ↓
[Step 2] スライド設計（デザインルール適用）
        ↓
[Step 3] pptx生成
        ↓
再現性の高いスライドデッキ
```

---

## Step 1: 入力の受け取り方

ユーザーからの入力は以下のどれでも対応する:

- **音声起こしテキスト**: 話し言葉のまま受け取り、構造化する
- **箇条書きメモ**: キーメッセージのリスト
- **1文の指示**: 「〇〇についての5枚スライドを作って」

### 入力を受け取ったら最初にやること

1. **スライド枚数と構成を提示して確認**（デフォルト: 表紙1枚＋本文3〜5枚＋まとめ1枚）
2. **テーマ/用途を確認**（商談用・社内共有・登壇用など）
3. 確認後、設計に進む

---

## Step 2: スライド設計ルール

### レイアウト原則（AIらしさを消す）

**やること:**
- 1スライド1メッセージ（本文は最大3行まで）
- タイトルは体言止めか短い動詞句（「〜について」禁止）
- 余白を意識的に使う（コンテンツは画面の60%以内に収める）
- フォントは統一（見出し: 太字・本文: レギュラー、サイズ差は明確に）

**やらないこと:**
- グラデーション背景（単色か白のみ）
- 丸角ボックスの多用（角丸は使うなら全スライドで統一）
- アイコン＋テキストの羅列（3つ以上は表形式に変換）
- 箇条書きの入れ子（2階層まで）

### カラーパレット戦略

ユーザーが指定しない場合:
- **ビジネス/商談用**: ネイビー＋白＋アクセントカラー1色
- **社内共有/カジュアル**: ライトグレー＋ブランドカラー
- **登壇/外部発表**: 黒背景＋白テキスト（高コントラスト）

パレットを決めたら、スライド全体で必ず一貫して使う。

### スライド構成テンプレート

```
[表紙]
- タイトル（大・中央）
- サブタイトル（小）
- 日付・発表者名（オプション）

[課題/背景スライド]
- 見出し: 問いかけ形式（例: 「なぜ今〇〇が必要か」）
- 本文: 3点以内の事実
- ビジュアル: 数字・グラフ優先

[解決策/提案スライド]
- 見出し: 動詞で始まる（例: 「〇〇で解決する」）
- 本文: 1〜2文の説明
- ビジュアル: フロー図・比較表

[まとめ/CTA]
- 見出し: 「Next Steps」「アクション」
- 本文: 3点以内の具体的アクション
```

---

## Step 3: pptx生成

スライド設計が決まったら、`pptx` スキルを使って生成する。

**pptxgenjs方式（テンプレートなし）の場合:**

```
pptxスキルの pptxgenjs.md を参照して生成
- マスタースライドで色・フォントを統一定義
- 各スライドはマスターを継承
- テキストボックスの位置は数値で固定（毎回同じレイアウト）
```

**既存テンプレートがある場合:**
```
pptxスキルの editing.md を参照
- テンプレートのスタイルを維持
- コンテンツのみ差し替え
```

---

## Cowork特有のプロンプト設計

Claude Coworkでスライドを作る際は、以下のプロンプト構造が再現性を高める:

### システムプロンプト（Coworkに設定推奨）

```
あなたはスライドデザイン専門のアシスタントです。
以下のルールを必ず守ってください:
- 1スライドに伝えることは1つだけ
- テキストは最小限（1スライド3行以内）
- デザインはシンプル・フラット（グラデーション・過度な装飾禁止）
- カラーパレット: [ここにユーザーのブランドカラーを記入]
- フォント: 見出しはBold, 本文はRegular
```

### ユーザーターン例

```
入力: 「アポドリの新機能紹介、5枚で。営業担当向け」

Claude の応答フロー:
1. 構成案を提示（表紙・課題・機能説明・効果・CTA）
2. 各スライドのタイトルと本文案を一覧で提示
3. 確認後 pptx 生成
```

## 品質チェックリスト

生成後に確認:
- [ ] 全スライドのフォント・サイズが統一されているか
- [ ] 1スライドのテキスト量は3行以内か
- [ ] カラーパレットが全スライドで一貫しているか
- [ ] タイトルが体言止めか短い動詞句か
- [ ] 余白が十分にあるか（コンテンツは画面の60%以内）
