# ネーミング・クラフト

プロダクト名・サービス名・ブランド名・社名などの命名を支援するスキル。

## ネーミングの原則

良いネーミングは **① 意味 → ② 語感 → ③ 独自性** の3つが揃っている。意味で方向性を定め、音で当てにいき、独自性で成立させる。この順番が重要で、いきなり「響きが良い言葉」から入ると意味が後付けになり薄くなる。

---

## ① 意味：ネーミングの出発点

意味とは「どんな体験価値を提供するのか」「どんな振る舞いをするものなのか」という要件のこと。

### 基本軸：デファクトスタンダードになれるか

ネーミングで最も重視すべきは、将来的にその分野を象徴する言葉になりうるかどうか。

「物を売る」を "メルカリする" と言うように、顧客にとって使いやすく思い出しやすい言葉であること。ビジネスの文脈でも、同じ領域だけでなく外からその領域を語る際に自然と想起される言葉であること。顧客とビジネスの両軸で使いやすいことが、デファクトスタンダードにつながる。

具体的に自問すべきこと：
- 友達に口頭で説明するとき、その名前を自然に言えるか？
- 業界ニュースの見出しに載ったとき、しっくり来るか？
- 「〇〇する」「〇〇的な」と動詞化・形容詞化できるか？

### ブランド文脈軸：事業の姿勢とネーミングの一致

事業やブランドが目指す姿勢・ポジショニングとネーミングが一致しているかを確認する。

例えば、アダルト領域のプロダクトであっても「健全に、明るく開かれたエンタメにしたい」という方針があるなら、アンダーグラウンドな印象を与えるネーミングは避ける。逆に「尖った印象で差別化する」方針なら、多少挑発的な語感もありえる。

事業の姿勢を確認するために以下を聞く：
- このプロダクト/サービスの社会的な立ち位置をどう定義したいか？
- ユーザーがこの名前を人前で口にするとき、どんな気持ちであってほしいか？
- 避けたいイメージ・連想はあるか？

### ターゲット文脈軸：使われる場で自然か

ターゲット顧客の文化圏で自然に使えるかも重要な判断基準。

- 会話の中で自然に出てくるか
- その文化圏でポジティブに受け入れられるか
- 競合のネーミング傾向を踏まえたとき、どう差別化するか

**例：競合との差別化の考え方**
競合が「FAKKU」のようにスラング由来で強い印象を取りに行っている場合、自分たちは王道の言葉で広く浸透していく方向を選ぶ、という判断ができる。相手を否定するのではなく「自分たちの立ち位置はどこか」を定めるための判断基準として使う。

## ② 語感：音で当てにいく

意味を定めたら、語感の良いワードを連想ゲーム的に展開する。ここでは「音」を重視する。

### 連想の展開方法

意味から導いた印象に合うワードを出し、それを音として当てはめていく。

**例：「ねおめろ」の場合**
1. 意味の整理：アダルトAIキャラチャット → 新時代エンタメ × アダルト
2. 音の連想：新時代（neo / new / era…）→ アダルト感のある音（えろえろ、ぺろぺろ → eoeo）→「ねおめろ」
3. 音として成立するか判定：口に出したとき言いやすい、4音で覚えやすい → OK

### 音の判断基準

- **言いやすさ**: 口に出して3回言ってみる。引っかかる音がないか
- **覚えやすさ**: 2〜5音節が理想。長すぎると略される（略された形も想定する）
- **響きの印象**: 硬い音（k, t, g）は力強さ、柔らかい音（m, n, r）は親しみやすさ
- **多言語での響き**: グローバル展開を想定する場合、英語・中国語・韓国語話者が発音しやすいか

### 連想を広げるときのテクニック

- 同義語・類語の横展開（neo → nova, prime, next, dawn…）
- 音韻の類似展開（メロ → モロ、マロ、ミロ、メル…）
- 合成語の実験（2つの語を組み合わせて造語にする）
- 語源への遡り（ラテン語・ギリシャ語に遡ると独自性が上がりやすい）
- 既存語のもじり（Spotify = spot + identify、Pinterest = pin + interest）

## ③ 独自性：その名前で戦えるか

最後に、そのネーミングが独自性を持って成立するかを確認する。

### チェックリスト

- **商標**: 同一・類似の商標が既に登録されていないか（特許庁J-PlatPat、USPTO、WIPO等で確認を推奨）
- **ドメイン**: .com / .app / .io 等の主要ドメインが取得可能か
- **SNSハンドル**: Twitter/X、Instagram等で同一ハンドルが取れるか
- **既存の強い意味との衝突**: 政治・宗教・差別・ネガティブワードとの衝突がないか。多言語でのチェックも忘れずに
- **検索の独自性**: Googleで検索したとき、既存の強い意味に埋もれないか
- **口にしやすさ**: 人前で言うとき恥ずかしくないか、電話で伝えられるか

### 最終判断

「その名前で戦えるか」が最終判断基準。具体的には：
- プレスリリースの見出しに入れてしっくり来るか
- 投資家へのピッチで堂々と言えるか
- ユーザーが友達に「〇〇っていうサービスがあってさ」と薦めるシーンを想像できるか

## 出力フォーマット

ネーミング案を出すときは、以下の形式で提案する。

1つの案につき：
- **名前**: 提案する名前
- **意味の説明**: なぜこの言葉を選んだか（意味軸の根拠）
- **語感の説明**: 音としてなぜ良いか（語感軸の根拠）
- **独自性の所感**: 商標・ドメイン等の初期所感（正式な調査は別途推奨）
- **想定される使われ方**: 「〇〇で読む」「〇〇する」のように実際に使われるイメージ

最低3案、できれば5〜7案を提示し、ユーザーの反応を見て方向性を絞り込む。

---

## ワークフロー

1. **ヒアリング**: 事業内容、ターゲット、ブランド姿勢、避けたいイメージ、競合の名前を聞く
2. **意味の軸を定める**: ヒアリングを元に、ネーミングが満たすべき意味の要件を整理する
3. **連想の展開**: 意味の軸から語感の良いワードを広く連想し、候補を出す
4. **絞り込み**: 独自性チェックを経て、有力な候補を3〜7案に絞る
5. **提案とフィードバック**: ユーザーに提示し、反応を見て方向性を調整する
6. **ブラッシュアップ**: フィードバックを元に候補を磨く。必要に応じて1から再展開する

## タイトル・コピーライティング設計

ネーミングだけでなく、記事タイトル、キャッチコピー、サービスの見出しなど「短いテキストで価値を伝える」場面でも本スキルの考え方が応用できる。以下はnoteの有料記事500件の分析から導かれた知見を統合したもの。

### 購入・クリックされるタイトルの3要素

ネーミングの「意味・語感・独自性」と同様に、タイトルにも3つの共通特徴がある。

1. **具体性**: 「3ステップ」「〇〇の始め方」など、内容のイメージが一発で伝わる。抽象的な言葉だけで構成されたタイトルは弱い。
2. **読者のメリットが明確**: 「初心者でも実践できる」「作業効率が10倍になる」など、読み手にとっての価値が明示されている。
3. **トレンド性**: 生成AI、副業、SNSなど、今関心の高いテーマを自然に含んでいる。

### タイトル設計の4つのポイント

タイトルを考える際は、以下の4つの問いに答える形で設計する。ネーミングの「意味」を定めるプロセスと本質的に同じ。

1. **誰に向けた内容か？**（例：副業初心者向け）
2. **どんな価値があるか？**（例：3ヶ月で初収益）
3. **どんな内容か？**（例：文章術の基本）
4. **どう伝えるか？**（例：体験談、3ステップで解説）

この4つが揃うと「自分のことだ」と思わせる具体性と、「読む価値がある」と思わせるメリットが同時に伝わる。

### 避けるべきタイトルパターン

以下のパターンは購入・クリックにつながりにくい。ネーミングで言う「独自性がない」「意味がぼやける」状態と同じ。

- **内容が曖昧**: 「最近思ったこと」
- **対象が不明確**: 「自分なりの考え方」
- **抽象語だけ**: 「幸せとは何か」
- **一般的すぎる**: 「ブログを書いてみた」
- **メリットが不明**: 「やってみた記録」

### 改善の型

弱いタイトルを強くする際の変換パターン：

- **プロセスと数字を入れて具体化**: 「ブログを書いてみた」→「はじめてのブログ開設体験記｜書き出すまでにやった3つのこと」
- **ターゲットを限定する**: 「自分なりの働き方」→「副業3年目で見えてきた、会社員と並行する仕事術」
- **得られる成果・メリットを明確にする**: 「ChatGPTの使い方」→「ChatGPTで仕事が10倍速に｜実際に役立った活用法5選」

### ネーミングとの接続

ネーミング（サービス名・ブランド名）とタイトル/コピーは異なるが、根底の原則は共通。

- **意味を定める** → タイトルでは「誰に・何を・どう」を定める
- **語感で当てる** → タイトルでは「リズム・テンポ・読みやすさ」を調整する（数字や区切り記号の活用）
- **独自性を確認する** → タイトルでは「検索したとき他と被らないか」「SNSでシェアされたとき目を引くか」を確認する

タイトルやコピーの依頼を受けた場合も、このスキルの① 意味 → ② 語感 → ③ 独自性の順番で設計すること。
