# Virtual Team Builder

Claude Codeで仮想の10部門経営チームを構築し、多角的なビジネス意思決定を支援するスキルです。

## このスキルが解決すること

一人で事業判断をすると視野が偏りがちです。このスキルは、マーケ・セールス・エンジニア・法務など10部門の専門家エージェントを定義し、それぞれの視点から意見・懸念・提案を出せる「仮想経営チーム」をClaude Code上に構築します。

## ワークフロー

### Step 1: 事業コンテキストの収集

ユーザーに以下を確認する（すべて任意、わかる範囲でOK）:

1. **事業内容**: どんなビジネス・プロダクトか？
2. **フェーズ**: スタートアップ初期 / 成長期 / 大企業 など
3. **直面している課題や意思決定**: 今、何を決めたいのか？
4. **チームの規模感**: 実際に何人くらいのチームか？
5. **特に重視したい部門**: （例：法務を厚くしたい、CFOの視点が重要 など）

情報が少なくても構わない。足りない部分は合理的に補完して進める。

### Step 2: 10部門エージェントの定義を生成

以下の10部門を基本構成とし、事業コンテキストに合わせてカスタマイズする:

| # | 部門 | ロール名（英語） |
|---|------|----------------|
| 1 | マーケティング | Chief Marketing Officer (CMO) |
| 2 | セールス | VP of Sales |
| 3 | エンジニアリング | CTO / VP of Engineering |
| 4 | プロダクト | Chief Product Officer (CPO) |
| 5 | ファイナンス | CFO / Head of Finance |
| 6 | 人事・組織 | Chief People Officer (CPO/CHRO) |
| 7 | 法務・コンプライアンス | General Counsel |
| 8 | カスタマーサクセス | VP of Customer Success |
| 9 | オペレーション | COO / Head of Operations |
| 10 | 経営企画・戦略 | Chief Strategy Officer (CSO) |

事業によっては部門名や優先順位を変える（例：FinTechなら「リスク管理」を追加、ECなら「ロジスティクス」を追加など）。

### Step 3: 各エージェントの定義を出力

各部門について以下の形式で出力する:

```markdown
## [部門名] Agent — [ロール名]

**役割・責任範囲:**
- [この部門が担う主な責任を3〜5点]

**判断の優先基準（Priority Framework）:**
1. [最優先で考慮すること]
2. [次に考慮すること]
3. [3番目に考慮すること]

**典型的な懸念・反論パターン:**
- [このロールが会議でよく指摘すること]
- [リスクとして上げやすいこと]

**他部門との連携ルール:**
- [密に連携すべき部門と、その理由]
- [衝突しやすい部門と、解決アプローチ]

**このエージェントに質問するのに適した場面:**
- [ユースケース例を2〜3点]
```

### Step 4: AGENTS.md / CLAUDE.md への組み込み方法を案内

生成したエージェント定義をClaude Codeで実際に活用する方法を説明する:

**方法A: AGENTS.md に保存する（推奨）**
```bash
# プロジェクトルートに保存
cat > AGENTS.md << 'EOF'
[生成したエージェント定義をここに貼り付け]
EOF
```
Claude Codeはプロジェクト内の `AGENTS.md` を自動的に読み込む。

**方法B: CLAUDE.md に追記する**
```bash
# 既存のCLAUDE.mdに追記
echo "" >> CLAUDE.md
echo "## Virtual Team Agents" >> CLAUDE.md
cat virtual-team.md >> CLAUDE.md
```

**方法C: 個別に呼び出す**
- `@CMO` のように部門名でメンションして意見を求める
- 「マーケティングの視点から見て〜」と自然言語で聞く

### Step 5: 活用例を提示

実際の使い方のイメージを示す:

```
ユーザー: 「この新機能のリリース判断をしたい」
→ 10部門それぞれの視点から意見を出してもらう

ユーザー: 「新しい採用計画について議論したい」
→ HR・Finance・Engineering・CSの視点から検討

ユーザー: 「競合他社のサービス開始にどう対応するか」
→ CMO・VP Sales・CSO・CPOの視点から対応策を出す
```

## 出力形式

最終的な成果物として以下を出力する:

1. **`virtual-team.md`** — 10部門エージェント定義ファイル（保存推奨）
2. **サマリー** — 各部門の一言説明と、今回の課題に特に重要な部門のハイライト
3. **今すぐ使える質問例** — 今回のユーザーの課題に対して、各エージェントに投げかけられる具体的な質問を3〜5個提示

## カスタマイズのヒント

- 10部門は固定ではない。「ユーザーリサーチ」「データサイエンス」「PR/広報」など、事業に応じて差し替えOK
- 各エージェントの「判断基準」は事業フェーズで変わる（初期スタートアップはスピード優先、大企業はリスク管理優先など）
- 日本語・英語どちらでも対応可。グローバルチーム想定なら英語で定義するのが自然
