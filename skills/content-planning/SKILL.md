# コンテンツ企画・運営スキル

コンテンツの企画から運営までをClaude Codeで一元管理するスキル。
非エンジニアでも「/content-planning」一発でコンテンツ計画が完了する。

---

## ワークフロー

### Step 1: 現状把握
ユーザーに以下を確認する（AskUserQuestion使用）：
- **チャネル**: X / note / YouTube / ブログ / その他
- **期間**: 今週 / 今月 / 四半期
- **テーマ領域**: AI / デザイン / ビジネス / プロダクト開発 / その他
- **頻度**: 毎日 / 週3 / 週1 / 月2 など

### Step 2: トレンド分析
1. WebSearchで指定テーマ領域の最新トレンドを調査
2. X上のバズ投稿・話題を確認（Chrome MCP使用可）
3. 競合・参考アカウントの投稿傾向を分析
4. 季節性・イベント（カンファレンス等）を考慮

### Step 3: テーマ提案
以下のフォーマットでテーマ候補を提案：

```markdown
## コンテンツ企画案（[期間]）

### テーマ1: [タイトル案]
- **チャネル**: X長文 / noteなど
- **切り口**: [ユニークな視点]
- **想定反応**: [なぜこのテーマが刺さるか]
- **優先度**: 高/中/低
- **関連トレンド**: [根拠となるトレンド]

### テーマ2: ...
```

### Step 4: 構成案作成
承認されたテーマについて、チャネルに応じた構成案を作成：

**X記事の場合** → `/x-article-writer` に引き継ぎ
**note記事の場合** → `/note-knowledge` または `/nyusha-entry` に引き継ぎ
**その他** → 本スキル内で構成案を出力

### Step 5: カレンダー化（オプション）
Notion連携が可能な場合、コンテンツカレンダーをNotionデータベースに作成：
- 投稿日
- テーマ
- チャネル
- ステータス（企画中/執筆中/レビュー中/公開済み）
- 担当者

## テーマ発想のフレームワーク

### 1. 体験ベース
「最近やったこと」「試してみたこと」から記事化
- 「〇〇を3日間使ってみた」
- 「〇〇をやめたら△△になった」

### 2. 知見ベース
業務で得た学びを一般化して共有
- 「〇〇で成果を出すために必要な3つのこと」
- 「〇〇の現在地と未来」

### 3. 逆張りベース
通説に対する異なる視点を提示
- 「〇〇は本当に必要か？」
- 「みんなが言わない〇〇の真実」

### 4. キュレーションベース
情報を整理して価値を生む
- 「〇〇に役立つツール10選」
- 「2024年の〇〇トレンドまとめ」

### 5. ストーリーベース
個人の経験をストーリーとして語る
- 「〇〇に入社して1年で学んだこと」
- 「失敗から学んだ〇〇」

## チャネル別の特性

| チャネル | 文字数 | 更新頻度 | 特徴 |
|---------|--------|---------|------|
| X（単独） | 〜280字 | 毎日可 | フック重視、即時性 |
| X（長文） | 〜2000字 | 週1-2 | 深い考察もOK |
| X（スレッド） | 3-7投稿 | 週1 | 段階的な展開 |
| note | 2000-5000字 | 月2-4 | 深い考察、SEO効果 |
| YouTube台本 | 5-15分分 | 月2-4 | 視覚的説明が必要なテーマ |

## 出力フォーマット

最終的なコンテンツカレンダーは以下の形式で出力：

```markdown
# コンテンツカレンダー [月]

| 日付 | チャネル | テーマ | ステータス | 備考 |
|------|---------|--------|-----------|------|
| 3/28 | X長文 | Claude Code×Figmaの実践知見 | 企画中 | トレンド連動 |
| 3/30 | note | AIネイティブ時代のデザイナー論 | 企画中 | 前回記事の続編 |
| 4/1 | X単独 | 新機能リリース告知 | 予定 | アポドリ |
```
