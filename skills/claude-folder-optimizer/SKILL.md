# .claude フォルダ最適化スキル

現在の.claudeフォルダを分析し、ベストプラクティスとの差分を提示して自動整理するスキル。

---

## 完全なフォルダ構造リファレンス

```bash
# グローバル設定
~/.claude/
├── CLAUDE.md           # グローバル指示（200行以内推奨）
├── rules/              # CLAUDE.mdが200行超えたら分割先
│   ├── tone.md         # 文体・口調ルール
│   ├── x-posting.md    # X投稿ルール
│   └── [担当別].md     # 担当ごとに分割
├── settings.json       # 権限・フック設定（3層管理）
├── skills/             # マニュアル実行スキル（テンプレート有り）
├── agents/             # 専門家エージェント（独立コンテキスト）
├── commands/           # カスタムコマンド
├── memory/             # 記憶システム
└── scheduled-tasks/    # 定期タスク

# プロジェクト設定
.claude/
├── CLAUDE.md           # プロジェクト固有指示
├── rules/              # プロジェクト固有ルール分割先
├── settings.json       # プロジェクト固有設定
├── settings.local.json # ローカル設定（gitignore推奨）
├── skills/             # プロジェクトスキル
└── launch.json         # Preview設定
```

## skills/ vs agents/ の使い分け

| | skills/ | agents/ |
|---|---|---|
| **呼び出し方** | `/skill-name` でマニュアル実行 | Claude が自律的に判断して呼び出す |
| **コンテキスト** | メインの会話コンテキストを共有 | 独立したコンテキスト（専門家として動く） |
| **テンプレート** | SKILL.md にテンプレート・手順を記述 | 役割・判断基準・ツール権限を定義 |
| **向いている用途** | 定型ワークフロー・記事執筆・レポート生成 | コードレビュー・セキュリティ監査・調査 |

**判断基準:** 「ユーザーが明示的に呼ぶ作業」→ skills、「Claude が状況判断して委任する専門家」→ agents

## settings.json の3層権限管理

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git diff*)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(cat .env*)",
      "Bash(cat *secret*)"
    ]
  }
}
```

| 設定 | 動作 | 使いどころ |
|---|---|---|
| `allow` | 確認なしで即実行 | 安全な読み取り・ビルドコマンド |
| `deny` | 絶対に実行しない | 破壊的操作・機密ファイルアクセス |
| 未設定 | 都度確認ダイアログ | デフォルト（慎重に扱うべきもの） |

**セキュリティ必須:** `.env` の読み取りは必ず `deny` に追加する。APIキー・秘密鍵の漏洩を防ぐため。

```json
"deny": ["Bash(cat .env*)", "Bash(cat *.env)", "Read(.env*)"]
```

## ワークフロー

### Step 1: 現状分析
現在の.claudeフォルダ構成をスキャンする（上記の完全な構造を参照）。

### Step 2: ベストプラクティスとの差分チェック

以下の項目をチェック：

#### CLAUDE.md / rules/
- [ ] プロジェクトの概要が記載されているか
- [ ] 技術スタック・フレームワークが記載されているか
- [ ] コーディング規約・命名規則が記載されているか
- [ ] ビルド・テスト・デプロイ方法が記載されているか
- [ ] 200行を超えている場合は `rules/` に分割できるか（tone.md / 担当別など）

#### skills/ と agents/ の整理
- [ ] スキル名がケバブケースか
- [ ] descriptionが具体的なトリガーワードを含むか
- [ ] skills/ と agents/ が適切に分類されているか（上記の使い分け基準を参照）
- [ ] 重複・類似スキルがないか
- [ ] 使われていない古いスキルがないか

#### settings.json のセキュリティ
- [ ] `.env` の読み取りが `deny` に設定されているか（必須）
- [ ] 破壊的コマンド（rm -rf、git reset --hard など）が適切に制限されているか
- [ ] 安全なコマンド（ビルド・テスト・読み取り）が `allow` に設定されているか
- [ ] hooksが適切に設定されているか

#### Memory
- [ ] MEMORY.mdが整理されているか
- [ ] 古い・不要な記憶ファイルがないか
- [ ] 記憶ファイルの形式が統一されているか

### Step 3: 最適化提案
差分をレポートとして出力：

```markdown
## .claude 最適化レポート

### 現在の構成
- グローバルスキル: N個
- プロジェクトスキル: N個
- CLAUDE.md: N行
- Memory: N件

### 改善提案

#### 優先度: 高
1. [具体的な改善提案]

#### 優先度: 中
1. [具体的な改善提案]

#### 優先度: 低
1. [具体的な改善提案]
```

### Step 4: 自動整理（承認後）
ユーザーの承認を得て、以下を実行：
- 重複スキルの統合
- CLAUDE.mdの整理
- 不要ファイルの削除
- フォルダ構造の最適化

## 新規プロジェクトセットアップ

新しいプロジェクトで `/claude-folder-optimizer` を実行すると、以下のテンプレートで初期設定を生成：

```
.claude/
├── CLAUDE.md           # プロジェクト概要・技術スタック・規約
├── settings.json       # 推奨権限設定
└── launch.json         # dev server設定（該当する場合）
```

CLAUDE.mdテンプレート:
```markdown
# プロジェクト名

## 概要
[プロジェクトの説明]

## 技術スタック
- フレームワーク:
- 言語:
- パッケージマネージャー:

## 開発コマンド
- 開発サーバー: `npm run dev`
- ビルド: `npm run build`
- テスト: `npm test`

## コーディング規約
- [規約]
```
