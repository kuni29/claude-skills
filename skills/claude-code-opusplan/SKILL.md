# Claude Code OpusPlan — モデル最適化ワークフロー

## 概要

Claude Codeには `/model opusplan` という隠しモードがある。

- **計画フェーズ** → Opus（最高精度の思考・アーキテクチャ設計）
- **実行フェーズ** → Sonnet（高速コーディング・ファイル編集）

を自動で切り替えることで、**品質を落とさずに速度を最大化**できる。

## 使い方

```
/model opusplan
```

Claude Codeのチャット欄で上記を実行するだけ。以降、
- 計画・設計・調査タスクはOpusが担当
- 実装・編集・ファイル操作はSonnetが担当

に自動切り替えされる。

## どんなときに使うか

### 使うべき場面
| シーン | 理由 |
|--------|------|
| 新機能のアーキテクチャ設計 | 複雑な依存関係・トレードオフの整理にOpusの思考力が必要 |
| バグの根本原因調査 | 問題の仮説立案・検証設計にOpusが有効 |
| 大規模リファクタリング | 変更影響の洗い出し段階でOpus、実行段階でSonnet |
| スキル・プロンプト設計 | SKILL.mdの構造設計にOpus、実際の記述にSonnet |
| 長大なタスク（30分以上想定） | 最初の計画品質がゴール品質を決める |

### 使わなくていい場面
- 単純なファイル編集・追記（Sonnet単独で十分）
- 既にやることが明確なコーディング
- 短時間で完結するタスク

## CLAUDE.mdへの組み込み推奨

重いタスクを定期的に扱う場合、CLAUDE.mdに以下を追記すると自動でリマインドできる：

```markdown
## モデル最適化
複雑なタスク（設計・調査・大規模変更）を開始する前に `/model opusplan` を使うこと。
単純な実行タスクには不要。
```

## 関連コマンド

- `/model` — 現在のモデルを確認・変更
- `/fast` — Fast モード（Opus 4.6高速出力）のトグル
- `/model sonnet` — Sonnetに固定（軽量タスク向け）
- `/model opus` — Opusに固定（最高品質モード）
