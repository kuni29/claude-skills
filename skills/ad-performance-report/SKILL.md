# アポドリ広告配信実績 自動分析レポート

## 概要

Algomaticのアポドリ広告配信実績スプレッドシートから最新データを取得し、広告代理店レポートと同等の粒度で自動分析レポートを生成する。

## データソース

### メインデータソース: Google Sheets
- **スプレッドシートID**: `1uujdzG1X35LErLEHCyRHPSxiF7T4uvkOMu7ULOgxd98`
- **全チャネルシート (gid=1957910060)**: 月次・週次・日次の広告配信実績
- **CPN分析・考察シート (gid=373635264)**: キャンペーン別分析と考察テキスト

### 全チャネルシートの構造
- **行1**: 目標値（オフラインソース306件、Web130件）
- **行2**: 合計ヘッダー
- **行3**: 列ヘッダー（年月, IMP, CLICK, CTR, CPC, 広告費(NET), 広告費(GROSS), ...）
- **行4**: 当月着予
- **行5-10**: 月次サマリー（20257=2025年07月, 20256=2025年06月, ...）
- **行12-13**: 目標（認知あり/認知なし）
- **行15**: 週次ヘッダー
- **行16-32**: 週次データ（認知広告配信週は黄色）
- **行36**: 日次ヘッダー
- **行37以降**: 日次データ

### 列構造（CSVの列位置）
列A: 広告種別（認知広告/検索広告）
列B: 配信タイプ（認知広告配信/指名CPN停止等）
列C: 年月コード
列D: 年月ラベル / 週開始日 / 日付
列E: 曜日（日次のみ）

**広告配信指標（列F-K）:**
- F: IMP, G: CLICK, H: CTR, I: CPC, J: 広告費(NET), K: 広告費(GROSS)

**獲得指標 - オフライン（列L-M）:**
- L: リード獲得（オフライン含む）, M: リード単価

**獲得指標 - Web全チャネル（列N以降）:**
- リード獲得, リード単価, 有効リード獲得, 有効リード率, 有効リード単価
- 商談獲得, 商談化率, 商談獲得単価
- 有効商談, 有効商談率, 有効商談単価
- 案件化, 案件化率, 案件単価
- 受注, 受注率, 受注単価

**GACV指標:**
- 乖離率（申込/GACV）, CV, CVR, CPA

**AD限定の獲得指標:**
- リード獲得_AD, リード単価_AD, 有効リード獲得_AD, 有効リード率_AD, 有効リード単価_AD
- 商談獲得, 商談化率, 商談獲得単価（ADのみ）
- 有効商談, 有効商談率, 有効商談単価（ADのみ）
- 案件化, 案件化率, 案件単価（ADのみ）
- 受注, 受注率, 受注単価（ADのみ）

**チャネル別:**
- 全チャネルセッション
- Organic Search
- youtube / pivot
- その他(organic, listing除外)

**Google CPN別:**
- 指名, 準指名, 一般_顕在, 一般_準顕在, 一般_潜在, カスタム_検索ユーザー
- 各CPNにつき: IMP, CLICK, CTR, CPC, 広告費(NET), 広告費(GROSS), リード, 有効リード, 有効リード率等

## 実行手順

### Step 0: 外部ツール接続チェック

以下のツールのMCPコネクタが接続されているか確認し、未接続のものがあればユーザーに接続を促す。
接続できないツールがあっても、スプレッドシートデータだけで基本レポートは生成可能。

1. **Google Analytics 4** - `search_mcp_registry`で「google analytics」「GA4」を検索
   - 接続済み: セッション数、チャネル別トラフィック、CVRのリアルタイムデータを補完取得
   - 未接続: `suggest_connectors`で接続を促すメッセージを表示
2. **Google Ads** - `search_mcp_registry`で「google ads」を検索
   - 接続済み: CPN別のIMP, CLICK, CPC, 広告費のリアルタイムデータを補完取得
   - 未接続: `suggest_connectors`で接続を促すメッセージを表示
3. **HubSpot** - `search_mcp_registry`で「hubspot」「CRM」を検索
   - 接続済み: リード〜受注の最新パイプラインデータを補完取得
   - 未接続: `suggest_connectors`で接続を促すメッセージを表示

未接続ツールがある場合の表示例:
```
データ補完ツールの接続状況:
- Google Analytics 4: 未接続 → 接続するとセッション・CVRのリアルタイムデータを分析に追加できます
- Google Ads: 未接続 → 接続するとCPN別の最新広告配信データを取得できます
- HubSpot: 未接続 → 接続するとリード〜受注の最新パイプラインデータを取得できます

現在はスプレッドシートのデータで基本レポートを生成します。
より詳細な分析が必要な場合は、上記ツールの接続をお勧めします。
```

### Step 1: モード判定

ユーザーの指示から以下を判定する:
- **モード**: `weekly`（デフォルト）/ `monthly` / `realtime`
- **出力先**: `terminal`（デフォルト）/ `notion` / `slack` / 複数
- **分析期間**: 指定があればそれに従う。なければweekly=直近週、monthly=直近月

### Step 2: データ取得

Chrome MCP（Claude in Chrome）を使ってスプレッドシートからデータを取得する。

1. `tabs_context_mcp` でタブコンテキストを取得
2. 新しいタブで以下のURLを開く:
   ```
   https://docs.google.com/spreadsheets/d/1uujdzG1X35LErLEHCyRHPSxiF7T4uvkOMu7ULOgxd98/export?format=csv&gid=1957910060
   ```
3. ダウンロードされたCSVファイルを `~/Downloads/` から読み取る
   - ファイル名パターン: `Algomatic様_アポドリ／広告配信実績 - 全チャネル.csv`

バックアップ手段: CSVエクスポートが失敗した場合は、スプレッドシートを直接開いてスクリーンショットから読み取る。

接続済みの外部ツールがある場合は、並行してデータを取得:
- GA4 MCP → セッション/トラフィックデータ
- Google Ads MCP → CPN別最新配信データ
- HubSpot MCP → パイプラインデータ

## 重要な分析ロジック

### 認知広告配信週の判定
- 全チャネルシートの列A（広告種別）に「認知広告」が入っている週 = 認知広告配信週
- 列Aが空または「検索広告」の週 = 非配信週

### 前期比の計算
- 増減率 = (当期値 - 前期値) / 前期値 × 100
- ポイント差 = 当期率 - 前期率（CTR, 有効リード率等の率指標）
- ゼロ除算の場合は「N/A」と表示

### 異常値検出
- 前週比 ±30%以上の変動があった指標をハイライト
- 有効リード率が平均から±10pt以上乖離した場合に注記

### CPN分析の評価基準
- 有効リード率: 40%以上=良好、30-40%=標準、30%未満=要改善
- リード単価: ¥15,000以下=良好、¥15,000-25,000=標準、¥25,000超=要改善
